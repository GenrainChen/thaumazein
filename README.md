# Thaumazein

διὰ γὰρ τὸ θαυμάζειν οἱ ἄνθρωποι καὶ νῦν καὶ τὸ πρῶτον ἤρξαντο φιλοσοφεῖν

*人类因惊奇而开始哲学思考。* —— 亚里士多德《形而上学》

---

三层架构智能感知-操作系统。核心原则是**下载式智能**：Hermes 创造所有智能，Bastion 执行智能产物，Terminal 转换物理信号。

```
┌───────────────────────────────────────────────────┐
│              高层 High Level                        │
│  高级编程语言 / 自然语言解读 · AI 推理 · 高级功能    │
├───────────────────────────────────────────────────┤
│              中间层 Middle Level                    │
│  汇编/C 语言 · 传感器/操作器型号抽象 · 驱动与通信    │
├───────────────────────────────────────────────────┤
│              底层 Physical Level                    │
│  物理实体 · 感知器 · 操作器 · 感知-操作一体设备      │
└───────────────────────────────────────────────────┘
```

## 产品线

| 产品 | 层次 | 职责 |
|---|---|---|
| **Hermes** 赫尔墨斯 | 高层 | 唯一面向人类的产品，唯一具备智能创造能力的节点。通过思维导图、流程图、编程、仿真四大工具设计系统 |
| **Bastion** 堡垒 | 中间层 + 高层 | 边缘程序执行中心，担当"包工头"：调度 Terminal、执行行为包、保障安全。有执行自主性，无创造自主性 |
| **Terminal** 终端 | 底层 | 物理信号转换节点，无智能。传感器与操作器之间的纯粹电磁-物理量转换 |

### Bastion 四种物理形态

| 形态 | 代号 | 设计哲学 |
|---|---|---|
| 陆地型 | Ground | 稳定可维护，性能优先 |
| 天空型 | Aerial | 轻量化、低功耗 |
| 潜水型 | Submarine | 耐压密封，通信受限 |
| 极端环境型 | Extreme | 可靠性至上 |

## 核心概念

### 行为包（Behavior Pack）

Hermes 将验证过的控制律、推理模型、安全规则打包为自包含、版本化的部署单元，下发 Bastion 本地执行。

```
behavior_pack/
├── manifest.json              # 元数据
├── models/                    # 预编译推理模型（ONNX/TFLite/TensorRT）
├── pipelines/                 # 信号处理流水线定义
├── control_laws/              # 控制律定义
├── safety_rules/              # 安全规则定义
├── schedules/                 # 执行调度
└── fallback/                  # 降级程序
```

### 三级控制时序

| 层级 | 频率 | 决策位置 | 示例 |
|---|---|---|---|
| 硬实时 | 100Hz–10kHz | Bastion 本地执行 | 电机 PID、传感器采样 |
| 软实时 | 1Hz–100Hz | Bastion 本地执行 | 路径规划、传感器融合 |
| 非实时 | <1Hz | Hermes 在线 | 任务规划、系统重构 |

### AI 四阶段

```
设计态（Hermes 创造） → 部署态（打包下发） → 执行态（Bastion 确定性执行） → 反馈态（Bastion 采集经验，Hermes 吸收）
```

### 降级梯次

Hermes 不可达时，Bastion 按预设梯次响应：

| 级别 | 触发条件 | Bastion 响应 |
|---|---|---|
| 链路降级 | 心跳延迟 > 2 倍正常 RTT | 继续运行，缓冲遥测，记录警告 |
| 链路丢失 | 30s 内无心跳 | 执行 Hermes 预设的降级程序 |
| 长期中断 | 链路丢失 > 30min | 执行最保守降级程序（安全保持或优雅停机） |

## 通信拓扑

```
Hermes ←──双向──→ Bastion (Hub) ←──双向──→ Terminal (Node)
  高层              中间层+高层            底层(物理)
```

- **Hermes ↔ Bastion**：行为包部署、遥测流、控制指令、经验数据、心跳（TLS 加密隧道）
- **Bastion ↔ Terminal**：寄存器级读写，HAL 抽象后为传感器轮询与操作器命令

## 安全

**功能安全（Safety）**：安全规则由 Hermes 设计并打包下发，Bastion 强制执行但不能修改。安全引擎独立于控制执行，即使主控制回路受损也能强制保护。

**信息安全（Security）**：纵深防御——行为包签名（Ed25519）、通信加密（TLS 1.3）、终端认证（注册 + HMAC）、固件 Secure Boot。每层独立承担安全责任。

## 设计哲学

基于四因说的认识论重构——四因不是世界的本体，而是人类认识世界、改造世界的基本思维模式。整个人类文明基于此而存在：

- **形式因**：概念的抽象——命名与界定（根本）
- **质料因**：形式因在空间维度的分化——"在哪里，由什么组成"
- **动力因**：形式因在时间维度的分化——"如何变化"
- **目的因**：人的选择——可选，决定验证路径（基于流程→动力因，基于数据→质料因）

AI 是人类的语言模块，桥接四因思维与计算机语言。Thaumazein 的职能是映射：将四因设计落实到程序（Hermes），再落实到物理实体（Bastion）。不需要物理实体时不需要 Bastion。

设计原则：形式因优先、拥抱抽象化、目的因自觉、形式因根本、简约惊奇、思维到实在的映射。

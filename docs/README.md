# Thaumazein 文档

本文档体系分为四层：方法论的哲学框架 → 架构设计的系统骨架 → 详细设计的工程规格 → 技术探索的研究支撑。每一层为下一层提供概念基础，下一层为上一层提供实现细节。

## 阅读路线

```
方法论 ──→ 架构设计 ──→ 详细设计（7 份） ──→ 技术探索（参考）
 哲学基础     系统骨架      工程规格           研究支撑
```

### 1. [方法论](方法论.md)

**起点。** 四因说认识论重构——四因是人类认识世界、改造世界的基本思维模式。定义人的角色（四因思维的践行者）、AI 的角色（人类的语言模块：翻译与搜索）、Thaumazein 的角色（从思维到实在的映射）。所有后续文档的概念基础源于此。

### 2. [架构设计](架构设计.md)

**方法论如何落地为系统。** 三层架构（Hermes / Bastion / Terminal）、产品线定义、通信拓扑、多节点协同、执行自主性边界。"下载式智能"原则、行为包机制、三级控制时序、降级梯次——均在方法论框架下展开。

### 3. 详细设计

架构设计拆解为工程规格，按关注面独立成文。无严格先后顺序，可按需阅读。

- **[模块设计](详细设计/模块设计.md)** — 架构如何拆解为代码模块。三大产品线的完整目录结构（hermes/、bastion/、terminal/）、行为包生命周期、各模块职责边界
- **[AI 融合架构](详细设计/AI融合架构.md)** — AI 如何融入三层架构。AI 四阶段（设计态→部署态→执行态→反馈态）的能力边界、知识流细节、Hermes AI 内部模块结构
- **[通信设计](详细设计/通信设计.md)** — 层间如何通信。Hermes-Bastion 通信接口、Bastion-Terminal 寄存器级操作、帧格式定义（含 Terminal 注册与 HMAC 认证）、数据模型（manifest schema、遥测格式）
- **[功能安全设计](详细设计/功能安全设计.md)** — 系统如何应对异常。功能安全（Safety）等级、降级梯次细化（按物理形态差异化）、安全规则类型、节点故障分类与响应策略
- **[信息安全架构](详细设计/信息安全架构.md)** — 系统如何防御威胁。信息安全（Security）纵深防御、认证体系、加密方案、威胁模型、固件 Secure Boot
- **[技术选型](详细设计/技术选型.md)** — 每层用什么技术及为什么。Hermes 框架、AI 模型、Bastion 引擎、Terminal 固件、通信协议的技术选型依据与适用场景
- **[星际接口](详细设计/星际接口.md)** — 预留设计，当前不实现。面向深空通信场景的接口预留

### 4. 技术探索

独立的研究报告，支撑方法论中的概念论述，非工程规格。

- **[语言模块≠思考模块](技术探索/语言模块≠思考模块：大语言模型认知局限与突破路径的深度研究报告.md)** — 论证"AI 是语言模块而非思考模块"的科学依据。认知科学证据（DMN、失语症、顿悟神经机制）、当前 LLM 的认知局限、迈向真正思考模块的前沿架构（GWT、顿悟计算、DIME）

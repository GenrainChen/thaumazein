# 功能安全设计（Safety）

本文档覆盖**功能安全（Safety）**。信息安全（Security）见 [信息安全架构](信息安全架构.md)。

## 安全原则

所有安全规则由 Hermes 设计并打包在行为包中。Bastion 的安全引擎强制执行这些规则，但**不能修改**它们。安全规则在架构上独立于控制执行，即使主控制回路受损也能强制执行。

### 安全规则的层级

```
┌─────────────────────────────────────────┐
│  物理硬保护（Terminal 硬接线）            │  ← 不可绕过，纯硬件
├─────────────────────────────────────────┤
│  Safety 引擎（Bastion safety/）          │  ← 独立于 engine/，可覆盖控制输出
├─────────────────────────────────────────┤
│  控制律约束（行为包 control_laws/）       │  ← 正常范围内的软约束
├─────────────────────────────────────────┤
│  任务级策略（行为包 schedules/）          │  ← 操作边界
└─────────────────────────────────────────┘
  优先级：从上到下递减，上层可覆盖下层
```

- **物理硬保护**：Terminal 的硬件级保护（熔断器、限流电路、机械限位）。不依赖任何软件，无法被软件绕过
- **Safety 引擎**：Bastion 的 safety/ 模块，独立于 control_exec/ 运行。可强制覆盖控制输出（如紧急制动），不受控制律影响
- **控制律约束**：行为包中的控制律自带软约束（如速度限幅、加速度限幅）
- **任务级策略**：schedules/ 中的操作边界（如工作空间限制、时间段限制）

### 安全规则的类型

| 类型 | 说明 | 示例 |
|---|---|---|
| 硬阈值 | 物理量超过阈值立即触发保护动作 | 电机电流 > 10A → 紧急断电 |
| 软阈值 | 物理量进入警告区间，触发降级但不停机 | 电池温度 > 60°C → 降低功率 |
| 时序约束 | 操作必须在规定时间内完成 | 响应命令超时 → 标记终端故障 |
| 状态互斥 | 某些状态不可同时存在 | "正在移动"与"正在维护"互斥 |
| 资源预留 | 为安全动作预留必要的计算/通信资源 | Safety 引擎占用独立 CPU 核心和通信通道 |

### 安全规则的验证

Hermes 在设计态对安全规则进行以下验证（由 safety_auditor 模块执行）：

1. **完整性检查**：每个执行器和关键物理量是否有对应的安全规则覆盖
2. **冲突检查**：安全规则之间是否存在矛盾（如一条要求升温，另一条要求降温）
3. **可达性检查**：安全规则的保护动作是否可达（执行路径上是否有其他阻塞）
4. **仿真验证**：在仿真中注入故障，验证安全规则能否正确触发和保护

## 降级梯次

当 Hermes 不可达时，Bastion 按 Hermes 预设的降级梯次响应：

| 级别 | 触发条件 | 持续时间 | Bastion 响应 |
|---|---|---|---|
| 正常 | 心跳正常 | - | 全功能运行 |
| 链路降级 | 心跳延迟 > 2 倍正常 RTT | 瞬态 | 继续当前程序，缓冲遥测，记录警告，提高心跳频率 |
| 链路丢失 | N 秒内无心跳（默认 30s，可配置） | 持续 | 按 Hermes 预设的降级程序执行（见下方降级程序） |
| 长期中断 | 链路丢失 > M 分钟（默认 30min，可配置） | 延长 | 执行最保守的降级程序（通常为安全保持或优雅停机） |
| Hermes 恢复 | 心跳恢复 | - | 上报累计状态，同步行为包版本，恢复正常运行 |

**关键规则**：所有降级程序由 Hermes 编写、仿真验证、打包下发。Bastion 不自创应急方案。

**按物理形态的推荐默认参数**：

| 参数 | Ground | Aerial | Submarine | Extreme |
|---|---|---|---|---|
| 链路丢失判定 | 30s | 3s | 60s | 60s |
| 长期中断判定 | 30min | 5min | 60min | 120min |
| 链路丢失默认动作 | safe_hold | graceful_stop（返航/降落） | safe_hold | emergency_stop |
| 长期中断默认动作 | graceful_stop | emergency_stop（最小功耗） | graceful_stop | emergency_stop |

- **Aerial**：空中平台对通信中断容忍度最低，需快速判定并执行返航或降落。3s 是平衡误判（瞬态丢包）和安全（避免失联坠落）的经验值
- **Submarine**：水声通信本身有数秒延迟，30s 可能过于敏感。60s 留出水声通信的正常波动空间
- **Extreme**：极端环境中降级应更保守——宁可提前停机也不冒险运行。长时间等待恢复是因为极端环境的恢复本身就需要更长时间

### 降级程序

降级程序存储在每个行为包的 fallback/ 目录中，与行为包版本绑定：

```
fallback/
├── degraded.json         # 链路降级策略
├── lost.json             # 链路丢失策略
├── prolonged.json        # 长期中断策略
└── recovery.json         # Hermes 恢复后的状态同步策略
```

每个策略文件定义：

```json
{
  "level": "lost",
  "actions": [
    {
      "type": "control_mode",
      "target": "control_law_id",
      "mode": "safe_hold",
      "params": {}
    },
    {
      "type": "telemetry",
      "action": "buffer",
      "max_size_mb": 100,
      "overflow": "drop_oldest"
    },
    {
      "type": "schedule",
      "action": "suspend_non_critical"
    }
  ],
  "terminal_handling": {
    "sensors": "continue_sampling",
    "actuators": "hold_position"
  }
}
```

**降级控制模式**：

| 模式 | 说明 | 适用场景 |
|---|---|---|
| `continue` | 继续执行当前控制律，不接受新指令 | 短期中断，控制律足够安全 |
| `safe_hold` | 切换到预设的安全保持位置/状态 | 链路丢失，需要快速保护 |
| `graceful_stop` | 按预设曲线缓慢停机 | 长期中断，无法安全保持 |
| `emergency_stop` | 立即停止一切运动 | 安全规则触发或极端情况 |

## 节点故障处理

### Terminal 故障

```
Terminal 故障
    │
    ▼
Bastion HAL 检测通信异常（超时 / NACK / 数据异常）
    │
    ├── 标记 Terminal 为 faulty
    ├── safety/ 评估影响范围
    │   ├── 孤立故障 → 标记为降级运行，上报 Hermes
    │   └── 关键终端故障 → 触发对应安全规则
    ├── control_exec/ 调整控制策略（排除故障终端的数据/输出）
    └── 上报 Hermes → Hermes 更新系统模型
                        → 可能重新配置其他 Terminal 补偿
                        → 可能下发新的行为包（含修正后的控制律）
```

### Bastion 故障

```
Bastion 故障
    │
    ├── Terminal 层面
    │   Terminal 固件检测命令链路丢失
    │   → 按固件内置的安全策略行动
    │   → 通常为：操作器保持当前位置，传感器继续低频采样
    │
    └── Hermes 层面
        Hermes 检测 Bastion 心跳丢失
        → 告警人类操作员
        → 标记该 Bastion 及其 Terminal 为离线
        → 不可远程恢复 Bastion 硬件故障，需人工介入
```

## 故障分类

| 类别 | 特征 | 检测方式 | 恢复方式 |
|---|---|---|---|
| 瞬态故障 | 短暂出现后自行恢复（电磁干扰、瞬间过载） | 超时重试成功、数据恢复正常 | 自动恢复，记录日志 |
| 间歇故障 | 反复出现但不持续（接触不良、温度相关） | 故障频率统计超过阈值 | 上报 Hermes，可能降级运行 |
| 永久故障 | 持续存在（硬件损坏、固件崩溃） | 重试失败、自检不通过 | 标记故障，隔离故障节点，人工更换 |
| 级联故障 | 一个故障触发其他故障（电源异常导致多终端失效） | 多节点同时异常 | safety/ 执行全局安全策略，隔离受影响区域 |
